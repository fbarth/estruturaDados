\begin{Slide}{Motivação}
  \begin{itemize}
  \item Vetor:

    \begin{itemize}
      \item ocupa um espaço contíguo de memória
      \item permite acesso randômico aos elementos
      \item deve ser dimensionado com um número máximo de elementos
    \end{itemize}
  \end{itemize}

  \begin{center}
    \includegraphics[width=.9\textwidth]{figuras/fig-01.pdf}
  \end{center}

\begin{Slide}{Motivação}
\begin{itemize}
\item Estruturas de dados dinâmicas: crescem ou decrescem à medida que
  elementos são inseridos ou removidos.
\item Exemplo: \emph{listas encadeadas}.
\item Listas encadeadas são amplamentes utilizadas para implementar
  outras estruturas de dados.
\end{itemize}
\end{Slide}


\begin{Slide}{Listas Encadeadas}
  \begin{center}
    \includegraphics[width=1\textwidth]{figuras/fig-02.pdf}
  \end{center}

\begin{itemize}
\item sequência encadeada de elementos, chamados \emph{nós} da lista.
\item \emph{nó da lista} é representado por dois campos:

\begin{itemize}
\item a informação armazenada e
\item o ponteiro para o próximo elemento da lista
\end{itemize}

\newpage

\item a lista é representada por um ponteiro para o primeiro nó
\item o ponteiro do último elemento é NULL.
\end{itemize}
\end{Slide}

\begin{Slide}{Definição recursiva de lista}
Uma lista é:
\begin{itemize}
\item uma lista vazia, ou;
\item um elemento seguido de uma (sub-)lista.
\end{itemize}
  \begin{center}
    \includegraphics[width=.9\textwidth]{figuras/fig-07.pdf}
  \end{center}
\end{Slide}

\begin{Slide}{Exemplo: função recursiva para imprimir uma lista}
\begin{itemize}
\item se a lista for vazia, não imprima nada
\item caso contrário,
\begin{itemize}
\item imprima a informação associada ao primeiro nó, dada por $prim.getInfo()$ 
\item imprima a sub-lista, dada por $prim.getProx()$, chamando recursivamente a função.
\end{itemize}
\end{itemize}
\end{Slide}

\begin{Slide}{Igualdade de listas}
\emph{boolean listasIguais(Nodo l1, Nodo l2)}

Implementação não recursiva:

\begin{itemize}
\item percorre as duas listas usando dois ponteiros auxiliares:
\begin{itemize} 
\item se duas informações forem diferentes então as listas são diferentes.
\end{itemize}
\item ao terminar uma das listas ou as duas:
\begin{itemize}
\item se os dois ponteiros auxiliares são NULL então as duas listas
  tem o mesmo número de elementos e são iguais.
\end{itemize}
\end{itemize}
\end{Slide}

\begin{Slide}{Igualdade de listas}
\emph{boolean listasIguais(Nodo l1, Nodo l2)}

Implementação recursiva:

\begin{itemize}
\item se as duas listas dadas são vazias então são iguais
\item se não forem ambas vazias, mas uma delas é vazia, então são
  diferentes
\item se ambas não forem vazias, teste:
\begin{itemize}
\item se informações associadas aos primeiros nós são iguais e
\item se as sub-listas são iguais.
\end{itemize}
\end{itemize}
\end{Slide}

\begin{Slide}{Listas de tipos estruturados}
Lista de tipo estruturado:
\begin{itemize}
\item a informação associada a cada nó de uma lista encadeada pode ser
  mais complexa, sem alterar o encadeamento dos elementos;
\item as funções apresentadas para manipular listas de inteiros podem
  ser adaptadas para tratar listas de outros tipos.

\newpage

\item o campo da informação pode ser representado por um objeto para
  uma estrutura, em lugar da estrutura em si.
\item independente da informação armazenada na lista, a estrutura do
  nó é sempre composta por: 
\begin{itemize}
\item um objeto para a informação e
\item um objeto para o próximo nó da lista.
\end{itemize}
\end{itemize}
\end{Slide}
